  • resourcePromise(opts)

    This returns a promise with an underlying http call, it is a helper method to reduce the amount of duplicate code needed to query http resources and automatically handle any Http errors. See /docs/source/using-promises-resources.md

    Parameters
    ParamTypeDetails
    optsobject

    A mixed object which can either be a string representing the error message to be returned OR an object containing either: { success: successCallback, errorMsg: errorMessage } OR { success: successCallback, error: errorCallback } In both of the above, the successCallback must accept these parameters: data, status, headers, config If using the errorCallback it must accept these parameters: data, status, headers, config The success callback must return the data which will be resolved by the deferred object. The error callback must return an object containing: {errorMsg: errorMessage, data: originalData, status: status }
