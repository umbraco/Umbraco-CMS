navigationService (service in module umbraco.services )

Description

Service to handle the main application navigation. Responsible for invoking the tree Section navigation and search, and maintain their state for the entire application lifetime

Dependencies

  • $rootScope
  • $routeParams
  • $log
  • $location
  • dialogService
  • treeService
  • sectionResource

Methods

  • changeSection(sectionAlias)

    Changes the active section to a given section alias If the navigation is 'sticky' this will load the associated tree and load the dashboard related to the section

    Parameters

    • sectionAlias {string}

      The alias of the section

  • hideDialog()

    hides the currently open dialog

  • hideMenu()

    Hides the menu by hiding the containing dom element

  • hideNavigation()

    hides any open navigation panes and resets the tree, actions and the currently selected node

  • hideSearch()

    hides the search pane

  • hideTree()

    Hides the tree by hiding the containing dom element

  • load(source)

    Shows the legacy iframe and loads in the content based on the source url

    Parameters

    • source {String}

      The URL to load into the iframe

  • showDialog(args)

    Opens a dialog, for a given action on a given tree node uses the dialogServicet to inject the selected action dialog into #dialog div.umb-panel-body the path to the dialog view is determined by: "views/" + current tree + "/" + action alias + ".html"

    Parameters

    • args {Object}

      arguments passed to the function

    • args.scope {Scope}

      current scope passed to the dialog

    • args.action {Object}

      the clicked action containing name and alias

  • showMenu(event)

    Hides the tree by hiding the containing dom element. This always returns a promise!

    Parameters

    • event {Event}

      the click event triggering the method, passed from the DOM element

  • showSearch()

    shows the search pane

  • showTree(sectionAlias)

    Shows the tree for a given tree alias but turning on the containing dom element only changes if the section is different from the current one

    Parameters

    • sectionAlias {string}

      The alias of the section the tree should load data from

  • showUserDialog()

    Opens the user dialog, next to the sections navigation template is located in views/common/dialogs/user.html
