# Working with Media Images

This guide covers how to display media images in the backoffice. In almost all cases you should reach
for a ready-made component rather than fetching and rendering image URLs yourself — the components
handle authentication, HMAC-signed URLs, lazy loading and caching for you.

## Which component should I use?

| Component | Cropping / resizing | Transparency | Use when… |
| --- | --- | --- | --- |
| `<umb-thumbnail>` | ✅ server-side resize & crop | ✅ configurable | You want a sized thumbnail — cards, pickers, collections, block grid items. **Recommended default.** |
| `<umb-media-image>` | ❌ original file | ✅ | You need the original, full-size image without resizing. |

> `<umb-imaging-thumbnail>` is a deprecated alias of `<umb-thumbnail>`, kept for backwards compatibility
> and scheduled for removal in Umbraco 19. Use `<umb-thumbnail>` in new code.

## Displaying a thumbnail

Pass the media item's `unique` (its key) and the desired dimensions. The component requests a resized,
HMAC-signed URL and renders it, falling back to an icon when the item has no image (e.g. a PDF).

<Source
	dark
	code={`<umb-thumbnail
	.unique=\${mediaUnique}
	.width=\${300}
	.height=\${300}
	.mode=\${UmbImagingCropMode.MIN}
	alt="My image"
	loading="lazy"></umb-thumbnail>`}
/>

<Story of={ThumbnailStories.Default} />

## Transparency: the checkerboard background

By default, transparent areas of an image are shown over a checkerboard pattern. This is intentional in
media-management contexts — it signals to an editor that the image has transparency.

When the thumbnail represents **final content** (for example a block grid item that should look like the
rendered site), remove the checkerboard with the `--umb-thumbnail-background` custom property:

<Source dark code={`umb-thumbnail { --umb-thumbnail-background: none; }`} />

Because it is a CSS custom property, it inherits — set it on any ancestor and every thumbnail underneath
picks it up. For full control over the underlying `<img>` (e.g. `object-fit`, `border-radius`), use the
`img` part:

<Source dark code={`umb-thumbnail::part(img) { object-fit: cover; border-radius: 4px; }`} />

<Story of={ThumbnailStories.TransparentBackground} />

## Going lower level

If you need to render images in a way the components don't support, use `UmbImagingRepository` directly.
It batches and caches requests, and returns ready-to-use absolute URLs:

<Source
	dark
	code={`const repository = new UmbImagingRepository(this);
const { data } = await repository.requestResizedItems([mediaUnique], {
	width: 300,
	height: 300,
	mode: UmbImagingCropMode.MIN,
});
const url = data[0]?.url;`}
/>

You generally shouldn't need this — `<umb-thumbnail>` wraps exactly this flow, including loading state,
the fallback icon and reloading when the media item changes.
